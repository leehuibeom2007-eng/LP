#기말 프로젝트 2번 파트(장애물) 프롬프트 활용 기록
작성자: 장호인

## 1. 초기 기획 및 프롬프트 (폐기됨)
초기에는 위에서 아래로 물체가 떨어지는 방식의 게임을 구상하여 AI에게 해당 로직을 요청함.
사용 프롬프트: "Pygame을 이용해 위에서 아래로 떨어지는 장애물과 아이템 클래스를 `enemy.py`로 분리해서 작성해 줘."
- 결과: 코드를 완성했으나, 팀 회의 결과 횡스크롤(마리오 형태) 게임으로 메인 기획이 확정되어 기존 코드를 폐기하고 팀장 코드를 베이스로 전면 수정 결정.

## 2. 코드 모듈화 및 충돌 방지 설계
- 상황: 팀장의 메인 코드(main.py) 파일 하나에 3명이 동시에 작업하면 GitHub Merge 시 충돌(Conflict)이 발생할 것을 우려함. 장애물 클래스만 별도의 파일로 빼내어 작업하기로 결정.
- 사용 프롬프트: "팀원이 작성한 메인 코드에서 `class Obstacle:` 부분만 독립된 파일(`obstacle.py`)로 분리해 줘. 깃허브에서 코드를 합칠 때 충돌이 나지 않도록 라이브러리 임포트(import) 등 구조를 캡슐화해 줘."
- 결과: 메인 코드에 의존하지 않고 독립적으로 작동 및 테스트가 가능한 `obstacle.py` 파일을 성공적으로 구축함.

## 3. 장애물 기믹 및 디자인 고도화 (바이브 코딩)
- 상황: 분리된 장애물 코드의 시각적 요소가 부족하여, 1학년과 사회인이라는 각 스테이지의 테마에 맞춰 디테일과 난이도를 높이는 코드를 AI와 함께 구현함.
- 사용 프롬프트: 
  1. "입영통지서(Military)가 하늘에서 떨어질 때, 유저가 미리 대비할 수 있도록 떨어지기 직전에 바닥(GROUND_Y)에 빨간색 느낌표(!) 경고 마크가 깜빡이게 하는 로직을 추가해 줘."
  2. "초록색 네모였던 입영통지서를 실제 국방색 봉투처럼 보이게 만들고, 그 위에 '8/25', '35사단'이라는 텍스트를 작게 렌더링해 줘."
  3. "스테이지 2의 세금(Tax) 폭탄 장애물은 모서리를 둥글게(border_radius) 처리하고 텍스트를 노란색으로 강조해 줘."
